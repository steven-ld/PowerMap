# PowerMap v0.5.0

## Highlights

- Adds domain mappings for HTTPS services. Multiple domains share one local `127.0.0.1:443` listener and are dispatched by TLS SNI.
- Makes hosts-file ownership explicit and recoverable. A failed domain activation remains visible with an actionable error.
- Removes active `web_token` enforcement for this release. The configuration field remains reserved for forward compatibility, but the management UI and API are directly available.

## Domain Mapping

Domain mappings are available on macOS and Linux only. PowerMap must run with administrator privileges because it owns marked entries in the system hosts file and binds local port 443. Clients must send TLS SNI. Existing port mappings continue to work independently.

## Compatibility

Existing `access.web_token` values are preserved but ignored. No Docker environment variable or root-start token bootstrap is required. Existing unified and legacy configuration migration behavior is unchanged.

## Upgrade

Replace the executable with `v0.5.0` and restart PowerMap. Back up `powermap.toml`, `powermap.key`, and `powermap.credential.json` before rollback.
