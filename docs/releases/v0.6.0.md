# PowerMap v0.6.0

## Highlights

- Adds reliable managed upgrades from the Node page. Native macOS/Linux installs can download, verify, replace, and restart in place.
- Adds environment-aware upgrade guidance for Docker, systemd, Windows scheduled tasks, and managed macOS LaunchAgents.
- Publishes multi-architecture Docker images to `ghcr.io/steven-ld/powermap` and keeps a GitHub Release draft until binary assets and images are verified.

## Upgrade Safety

Downloaded archives are SHA-256 verified and safely unpacked. Native in-process updates retain the old binary during a startup gate; if the new process fails during startup, PowerMap restores and re-executes the old binary.

Windows updates stop the `PowerMap` scheduled task before replacing `powermap.exe`, then start it again. Docker deployments should use the generated version-pinned Compose command from the host.

## Compatibility

Existing `powermap.toml`, node identities, credentials, and mappings are retained. No configuration migration is required for v0.6.0.
