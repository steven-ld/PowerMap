# PowerMap v0.7.0

Release date: 2026-07-23

## Highlights

- The About page now owns update checks, release notes, runtime status, and connection diagnostics. A compact header notice appears when a newer stable release is available.
- Overview adds mapping-health counts, state-change alerts, and moves live metrics plus refresh controls out of the navigation area.
- Operators can download a redacted diagnostic report for support and incident triage. The report omits credentials and local paths.
- Mapping creation provides common service templates and bounded loopback port suggestions.
- Configuration exports are redacted by default. Full exports require an explicit confirmation in the console.

## Release And Deployment

The release workflow builds native archives for Linux, macOS, and Windows, verifies their checksums before publishing, and builds Docker images from the verified Release archives instead of compiling application source in the image build.

## Compatibility

Existing `powermap.toml`, node identity files, credentials, port mappings, domain mappings, and legacy single-tenant tokens continue to work without migration. A redacted export intentionally omits the access token; use a full export only when secure credential portability is required.
