# PowerMap v0.4.0

## Summary

PowerMap v0.4.0 unifies the former server and client programs into one
`powermap` executable. Start the same binary on every machine and use the
configuration plus the local console to choose what that machine exposes and
which remote node it accesses.

## Highlights

- One binary, one `powermap.toml`, and unified systemd, launchd, Windows, Docker,
  installer, and Release asset naming.
- Automatic startup migration from `powermap-server.toml` and
  `powermap-client.toml` when no unified config exists.
- A reorganized local console with separate Overview, Port mappings, Nodes, and
  Events pages. The Nodes page defaults to connecting a remote node; the
  Overview shows the connected peer IPs when direct addresses are available.
- TCP remains the default mapping mode. UDP and HTTP gateway mappings are
  available through Advanced protocols.
- UDP passthrough, host-based HTTP gateway routing, reverse mappings, mapping
  names, in-place edits, connection details, and import merge mode.

## Upgrade

1. Back up the configuration directory, especially `powermap.toml`,
   `powermap.key`, and `powermap.credential.json`.
2. Install the new `powermap` binary from the matching Release archive and
   verify its SHA-256 checksum.
3. Start `powermap` normally. If only legacy role configs are present, they are
   merged into `powermap.toml` automatically.
4. Confirm the console at `http://127.0.0.1:8088`, then copy the local share
   credential to any access devices that need it.

The migration removes legacy config files only after the new unified config is
written successfully. Old mappings without a `mode` field continue as TCP, and
the legacy single-tenant top-level token remains valid as the `default` client.

## Compatibility

| Area | v0.4.0 behavior |
|---|---|
| Executables | Release archives contain `powermap` only. |
| Legacy config | `powermap-server.toml` and `powermap-client.toml` migrate automatically on startup. |
| Existing mappings | Preserved; omitted `mode` is TCP. |
| Single-tenant token | Preserved and normalized to the `default` client. |
| Supported Release targets | Linux x86_64/aarch64, macOS Intel/Apple Silicon, Windows x86_64. |
| Rollback | Back up unified files first; old role-specific binaries do not read `powermap.toml`. |

## Verification

- `cargo fmt --all -- --check`
- `cargo clippy --all-targets -- -D warnings`
- `cargo test --all-targets`
- `cargo build --release --bin powermap`

Release archives are accompanied by SHA-256 checksums and are smoke-tested by
the release workflow with `powermap --help` after extraction.
